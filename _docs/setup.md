# Setup & Operations

## Prerequisites

- Go 1.23+
- A valid `talosconfig` file with access to your Talos cluster (generated by `talosctl`)

## Authentication

Talos uses mTLS for all API communication. The `talosconfig` file contains the client certificate, key, and CA certificate needed to authenticate with the cluster.

| Priority | Source | When used |
|---|---|---|
| 1 | `TALOSCONFIG` env var | If set |
| 2 | `~/.talos/config` | Default talosconfig location |

## Environment Variables

| Variable | Required | Default | Description |
|---|---|---|---|
| `TALOSCONFIG` | No | `~/.talos/config` | Path to talosconfig file |
| `TALOS_CONTEXT` | No | current context | Override the active talosconfig context |
| `TALOS_ENDPOINTS` | No | from talosconfig | Comma-separated endpoint overrides |
| `TALOS_NODES` | No | — | Default nodes to target when not specified per-tool |

## Local Installation

```bash
cd talos-mcp
cp .env.example .env
# Edit .env if needed
make tidy
make build   # → ./bin/talos-mcp
```

## Integration with Claude Desktop

```json
{
  "mcpServers": {
    "talos": {
      "command": "/absolute/path/to/talos-mcp/bin/talos-mcp",
      "env": {
        "TALOSCONFIG": "/Users/you/.talos/config",
        "TALOS_CONTEXT": "my-cluster",
        "TALOS_NODES": "192.168.1.10,192.168.1.11,192.168.1.12"
      }
    }
  }
}
```

## Available Tools

### Node Management

| Tool | Description |
|---|---|
| `talos_get_version` | Get Talos version info from nodes |
| `talos_reboot` | Reboot nodes (soft or powercycle) |
| `talos_shutdown` | Shut down nodes |
| `talos_upgrade` | Upgrade Talos installer image |
| `talos_get_mounts` | List mounted filesystems |

### Services

| Tool | Description |
|---|---|
| `talos_list_services` | List all services and their status |
| `talos_start_service` | Start a service |
| `talos_stop_service` | Stop a service |
| `talos_restart_service` | Restart a service |
| `talos_get_service_logs` | Get logs for a system service |

### System

| Tool | Description |
|---|---|
| `talos_get_memory` | Get memory usage statistics |
| `talos_list_processes` | List running processes |
| `talos_get_disks` | List physical disks |
| `talos_get_dmesg` | Get kernel ring buffer output |

### etcd

| Tool | Description |
|---|---|
| `talos_etcd_members` | List etcd cluster members |
| `talos_etcd_status` | Get etcd member status |
| `talos_etcd_alarm_list` | List active etcd alarms |
| `talos_etcd_defragment` | Defragment etcd storage |

## Node Targeting

Most tools accept a `nodes` parameter (comma-separated IP addresses). If omitted, the server falls back to `TALOS_NODES`. If that is also unset, the request targets whatever the talosconfig context specifies.

```
# Single node
nodes: "192.168.1.10"

# Multiple nodes
nodes: "192.168.1.10,192.168.1.11,192.168.1.12"
```
